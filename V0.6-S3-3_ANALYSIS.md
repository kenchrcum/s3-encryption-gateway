### A.3 AppendPart Fail-Closed Alignment ✅

**File:** `internal/api/upload_part_copy.go` (862 lines)

**Status:** ✅ COMPLETE (Phase E re-encrypt path at lines 806–856)

**Original bug** (described in plan §1.2 gap 3):
- Lines 806–813 (old) logged warning and continued when `AppendPart` failed
- Left Valkey state inconsistent; returned 200 OK despite backend write success but state unrecorded
- **Silent data loss:** subsequent `Complete` would miss the part or fail cryptically

**Fix implemented** (lines 828–856):
```go
// Line 828-829: Fail-closed comment explaining the design
// Persist the part record. Fail-closed: a state-store hiccup here must
// return 503 so the client retries, matching handlers.go:2730-2757.

// Lines 830–837: Construct PartRecord for Valkey state store
chunkCount := int32((int64(len(plaintextData)) + int64(crypto.DefaultChunkSize) - 1) / int64(crypto.DefaultChunkSize))
if appendErr := h.mpuStateStore.AppendPart(ctx, uploadID, mpu.PartRecord{ ... })

// Lines 838–844: Log error + record metrics (same as UploadPart path)
h.logger.WithError(appendErr).WithFields(logrus.Fields{ ... })
  .Error("mpu.append_part.failure: backend part written but state not recorded; returning 503 so client retries")
h.metrics.RecordS3Error(ctx, "AppendMPUPartState", dstBucket, "StateUnavailable")

// Lines 845–854: Emit audit event
if h.auditLogger != nil {
  h.auditLogger.Log(&audit.AuditEvent{
    EventType: audit.EventTypeMPUValkeyUnavail,
    ...
  })
}

// Line 855: Return fail-closed sentinel
return nil, 0, errMPUStateUnavailable
```

**Error sentinel** (line 636):
```go
errMPUStateUnavailable = fmt.Errorf("mpu state store unavailable")
```

**Handler translation** (lines 345):
```go
} else if errors.Is(strategyErr, errMPUStateUnavailable) {
  // Returns 503 ServiceUnavailable (via caller at handleUploadPartCopy)
```

**Idempotent retry semantics:**
- Client retries on 503
- Duplicate backend part from retry discarded by `CompleteMultipartUpload`'s ETag-set reconciliation
