# Implementation Plan: V0.6-SEC-1 Pluggable KeyManager Interface

## Overview
This task refactors the crypto system to use a pluggable `KeyManager` interface, enabling support for different key storage backends (in-memory, KMS, HSM) with clean separation of concerns.

---

## Architectural Approach

According to **"Learning Go, 2nd Edition"** (O'Reilly, 2024) and **"Hands-On Software Engineering with Golang"** (Packt, 2020), Go's interface-based design enables flexible, testable architectures. The approach follows these patterns:

1. **Minimal, focused interface** - Define only essential key management operations
2. **Adapter pattern** - Separate implementations for each backend
3. **Dependency injection** - Engine depends only on the interface, not concrete types
4. **Testing via mocks** - Interface makes unit testing straightforward

## Key Design Decisions

| Decision | Rationale |
|----------|-----------|
| **Minimal interface** | Easier to implement for HSM vendors; reduces breaking changes |
| **Factory pattern in config** | Enables configuration-driven adapter selection |
| **Mock-friendly interface** | Supports dependency injection for testing (cite: "Hands-On Software Engineering with Golang" - testing via mocks) |
| **Explicit zeroization docs** | Critical for security; prevents sensitive key leaks |

## Example Interface Structure (Conceptual)

```go
// KeyManager defines the interface for pluggable key management backends.
// All implementations MUST support zeroization of sensitive data.
type KeyManager interface {
	// GetKey retrieves the key with the given identifier.
	// Returns an error if the key is not found or the backend is unavailable.
	GetKey(ctx context.Context, keyID string) ([]byte, error)

	// RotateKey initiates key rotation by creating a new key version.
	// The old key remains available for decryption during transition period.
	RotateKey(ctx context.Context, keyID string) (string, error)

	// ListKeys returns all available key identifiers.
	ListKeys(ctx context.Context) ([]string, error)

	// DeleteKey marks a key for deletion (backend-specific behavior).
	DeleteKey(ctx context.Context, keyID string) error

	// Health returns the backend's current health status.
	Health(ctx context.Context) error
}
```
