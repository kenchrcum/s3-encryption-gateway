## 2. Detailed Design

### 2.1 KDF Params Metadata Field

A single new S3 metadata field is added to every newly-written object:

```
x-amz-meta-encryption-kdf-params
```

**Format:** a colon-delimited string that encodes the KDF algorithm and its
parameters in a future-proof way.

| KDF | Format | Example |
|-----|--------|---------|
| PBKDF2-SHA256 | `pbkdf2-sha256:<iterations>` | `pbkdf2-sha256:600000` |
| argon2id (future) | `argon2id:<time>:<memory_kib>:<threads>` | `argon2id:2:19456:1` |

**Design rationale:**
- Single field avoids multiple metadata entries and a second migration when argon2id arrives.
- Colon-delimited tokens are unambiguous (no injection risk; no `|` delimiter issue from V1.0-SEC-H01).
- The field is absent on all objects written before this change; the absent case is the legacy fallback trigger.

**Constant to add in `engine.go`:**

```go
// MetaKDFParams stores the KDF algorithm and parameters used to derive the
// per-object encryption key. Format: "pbkdf2-sha256:<iterations>" or
// "argon2id:<time>:<memory_kib>:<threads>".
// Absent on objects written before V1.0-SEC-H03.
MetaKDFParams = "x-amz-meta-encryption-kdf-params"
```

### 2.2 `KDFParams` Struct and Helpers (`internal/crypto/kdf.go`)

New file `internal/crypto/kdf.go`:

```go
package crypto

import (
    "errors"
    "fmt"
    "strconv"
    "strings"
)

// KDFAlgorithm identifies the key derivation function.
type KDFAlgorithm string

const (
    KDFAlgPBKDF2SHA256 KDFAlgorithm = "pbkdf2-sha256"
    KDFAlgArgon2id     KDFAlgorithm = "argon2id"   // non-FIPS only

    // Legacy sentinel: returned when MetaKDFParams is absent.
    // Treated as pbkdf2-sha256 with LegacyPBKDF2Iterations.
    LegacyPBKDF2Iterations = 100000
    DefaultPBKDF2Iterations = 600000
    MinPBKDF2Iterations     = 100000
)

// KDFParams holds the parsed parameters for a KDF stored in MetaKDFParams.
type KDFParams struct {
    Algorithm  KDFAlgorithm
    // PBKDF2 fields
    Iterations int
    // argon2id fields (zero when not argon2id)
    Time       uint32
    Memory     uint32 // KiB
    Threads    uint8
}

// FormatKDFParams serialises KDFParams to the MetaKDFParams wire format.
func FormatKDFParams(p KDFParams) string { ... }

// ParseKDFParams parses a MetaKDFParams value.
// Returns the LegacyPBKDF2Iterations PBKDF2 params when value is empty (absent).
func ParseKDFParams(value string) (KDFParams, error) { ... }

// DefaultKDFParams returns the KDFParams for newly-written objects
// using the configured iteration count.
func DefaultKDFParams(pbkdf2Iterations int) KDFParams {
    return KDFParams{Algorithm: KDFAlgPBKDF2SHA256, Iterations: pbkdf2Iterations}
}
```

### 2.3 Engine Struct Changes (`internal/crypto/engine.go`)

**Struct field addition:**

```go
type engine struct {
    password            []byte
    pbkdf2Iterations    int    // NEW: configurable, default DefaultPBKDF2Iterations
    // ... existing fields unchanged
}
```

**Constructor chain addition:**  
`NewEngineWithChunkingAndProvider` gains a final `pbkdf2Iterations int` parameter. All existing public constructors (`NewEngine`, `NewEngineWithCompression`, etc.) default to `DefaultPBKDF2Iterations = 600000`.

A new top-level constructor is also added to engine_options.go:

```go
// WithPBKDF2Iterations sets the PBKDF2 iteration count for the engine.
func WithPBKDF2Iterations(n int) Option {
    return func(e *engine) {
        if n >= MinPBKDF2Iterations {
            e.pbkdf2Iterations = n
        }
    }
}
```

**`deriveKey` rewrite:**

```go
// deriveKeyWithParams derives a key using the given KDFParams.
func (e *engine) deriveKeyWithParams(salt []byte, params KDFParams) ([]byte, error) {
    switch params.Algorithm {
    case KDFAlgPBKDF2SHA256:
        return pbkdf2.Key(sha256.New, string(e.password), salt, params.Iterations, aesKeySize)
    default:
        return nil, fmt.Errorf("unsupported KDF algorithm: %s", params.Algorithm)
    }
}

// deriveKey derives a key for new objects using the engine's configured iterations.
func (e *engine) deriveKey(salt []byte) ([]byte, error) {
    return e.deriveKeyWithParams(salt, DefaultKDFParams(e.pbkdf2Iterations))
}
```

**On encrypt — write metadata:**

```go
encMetadata[MetaKDFParams] = FormatKDFParams(DefaultKDFParams(e.pbkdf2Iterations))
```

This line is added at the same place as `MetaKeySalt`, `MetaIV`, etc. — in both the
non-chunked `encryptObject` path and the chunked `encryptChunked` path.

**On decrypt — read metadata with fallback:**

```go
// Read KDF params; absent → legacy 100k PBKDF2.
kdfParams, err := ParseKDFParams(expandedMetadata[MetaKDFParams])
if err != nil {
    return nil, nil, fmt.Errorf("failed to parse KDF params: %w", err)
}
key, err = e.deriveKeyWithParams(salt, kdfParams)
```

This replaces the current `e.deriveKey(salt)` call in **both** the non-chunked
decrypt path and the chunked decrypt path.

### 2.4 PasswordKeyManager Changes (`internal/crypto/password_keymanager.go`)

```go
type passwordKeyManager struct {
    password         []byte
    pbkdf2Iterations int   // NEW
    closed           bool
}

// NewPasswordKeyManager creates a KeyManager with the given PBKDF2 iteration count.
// Callers should pass cfg.Encryption.KDF.PBKDF2.Iterations.
func NewPasswordKeyManager(password []byte, pbkdf2Iterations int) (KeyManager, error) { ... }
```

`WrapKey` writes the iterations into the envelope's `Ciphertext` prefix or into
an auxiliary field. Because `KeyEnvelope.Ciphertext` is an opaque `[]byte` owned
by this provider, the simplest approach is to **prepend a 4-byte big-endian
iteration count** to the existing `salt||nonce||sealed` layout:

```
[4-byte BE iterations][salt(32)][nonce(12)][sealed(48+)]
```

`UnwrapKey` reads the 4-byte prefix; if the payload is shorter than
`4 + saltSize + nonceSize + tagSize`, it checks if it matches the old
`saltSize + nonceSize + tagSize` layout and assumes `100000` iterations for
backward compatibility.

This is the only place a new binary format is introduced. The outer `KeyEnvelope`
fields (Provider, KeyVersion, CreatedAt) are unchanged.

### 2.5 Config Changes (`internal/config/config.go`)

The KDF configuration is nested under a top-level `encryption.kdf` key. This keeps all key-derivation concerns in one place and allows a future `encryption.kdf.algorithm` selector to switch between PBKDF2 and argon2id without polluting the flat `encryption.*` namespace.

**New structs:**

```go
// KDFConfig is the top-level key-derivation configuration block.
// YAML key: encryption.kdf
type KDFConfig struct {
    // PBKDF2 holds PBKDF2-SHA256 parameters.
    // Active when encryption.kdf.algorithm == "pbkdf2-sha256" (the default).
    PBKDF2 PBKDF2Config `yaml:"pbkdf2"`
}

// PBKDF2Config holds parameters specific to PBKDF2-SHA256 key derivation.
// YAML key: encryption.kdf.pbkdf2
type PBKDF2Config struct {
    // Iterations is the PBKDF2-SHA256 iteration count used for new objects.
    //
    // NIST SP 800-132 (2023) minimum: 600,000 (OWASP current recommendation).
    // Legacy default prior to V1.0-SEC-H03: 100,000.
    // Objects encrypted with the old default are decrypted transparently via
    // the KDF-params metadata fallback; no manual intervention is required.
    // To actively upgrade old objects, run:
    //   s3eg-migrate --filter=kdf
    //
    // Minimum enforced: 100,000.
    // Recommended production value: 600,000 or higher (tune so derivation ≤ 100ms).
    Iterations int `yaml:"iterations" env:"ENCRYPTION_KDF_PBKDF2_ITERATIONS"`
}
```

**Updated `EncryptionConfig`:**

```go
type EncryptionConfig struct {
    Password            string           `yaml:"password" env:"ENCRYPTION_PASSWORD"`
    KeyFile             string           `yaml:"key_file" env:"ENCRYPTION_KEY_FILE"`
    PreferredAlgorithm  string           `yaml:"preferred_algorithm" env:"ENCRYPTION_PREFERRED_ALGORITHM"`
    SupportedAlgorithms []string         `yaml:"supported_algorithms" env:"ENCRYPTION_SUPPORTED_ALGORITHMS"`
    KeyManager          KeyManagerConfig `yaml:"key_manager"`
    ChunkedMode         bool             `yaml:"chunked_mode" env:"ENCRYPTION_CHUNKED_MODE"`
    ChunkSize           int              `yaml:"chunk_size" env:"ENCRYPTION_CHUNK_SIZE"`
    Hardware            HardwareConfig   `yaml:"hardware"`
    KDF                 KDFConfig        `yaml:"kdf"`   // NEW
}
```

**Config YAML shape (for documentation and Helm):**

```yaml
encryption:
  kdf:
    pbkdf2:
      iterations: 600000   # NIST SP 800-132 (2023) minimum; default from V1.0-SEC-H03
```

**Default in `DefaultConfig()`:**

```go
Encryption: EncryptionConfig{
    // ... existing defaults ...
    KDF: KDFConfig{
        PBKDF2: PBKDF2Config{
            Iterations: 600000,
        },
    },
},
```

**Validation in `config.Validate()`:**

```go
if cfg.Encryption.KDF.PBKDF2.Iterations < 100000 {
    return fmt.Errorf("encryption.kdf.pbkdf2.iterations must be >= 100000 (got %d)",
        cfg.Encryption.KDF.PBKDF2.Iterations)
}
if cfg.Encryption.KDF.PBKDF2.Iterations < 600000 {
    // Warn only — 100k is valid for migration periods.
    slog.Warn("encryption.kdf.pbkdf2.iterations is below NIST SP 800-132 (2023) recommendation of 600,000",
        "iterations", cfg.Encryption.KDF.PBKDF2.Iterations)
}
```

**Env-var wiring in `loadFromEnv()`:**

```go
if v := os.Getenv("ENCRYPTION_KDF_PBKDF2_ITERATIONS"); v != "" {
    if n, err := strconv.Atoi(v); err == nil && n >= 100000 {
        config.Encryption.KDF.PBKDF2.Iterations = n
    }
}
```

### 2.7 Migration Tool Changes

#### 2.7.1 New Object Class (`internal/migrate/classifier.go`)

```go
// ClassD_LegacyKDF: object is encrypted but has no MetaKDFParams field,
// meaning it was created before V1.0-SEC-H03 and uses the legacy 100k
// PBKDF2 iteration count. Needs re-encryption to pick up the new default.
ClassD_LegacyKDF  ObjectClass = iota
```

**Classification rule** (added to `ClassifyObject`):

```go
// After existing class checks that did not match (ClassModern reached):
// A modern object without MetaKDFParams is Class D — legacy KDF.
if meta[crypto.MetaKDFParams] == "" {
    return ClassD_LegacyKDF
}
// MetaKDFParams present — could still be "legacy value" if operator changed
// target iterations after writing. That is intentional and transparent.
return ClassModern
```

`NeedsMigration` is updated:

```go
case ClassD_LegacyKDF:
    return true
```

#### 2.7.2 New Filter (`internal/migrate/migrator.go`)

```go
FilterKDF MigrationClassFilter = "kdf"
```

```go
case FilterKDF:
    return c == ClassD_LegacyKDF
```

#### 2.7.3 New `Migrator` Fields

```go
type Migrator struct {
    // ... existing fields ...
    SourceIterations int  // PBKDF2 iterations of existing objects (default 100000)
    TargetIterations int  // PBKDF2 iterations for re-encrypted objects (default 600000)
}
```

In `Migrate()`, when `SourceIterations > 0` and `TargetIterations > 0`, the
migrator constructs the `SourceEngine` and `TargetEngine` internally using those
iteration counts (if they are not already caller-supplied):

```go
// Build source engine with legacy iterations for decryption.
if m.SourceEngine == nil && m.SourceIterations > 0 {
    m.SourceEngine, err = crypto.NewEngineWithOpts(password, nil,
        crypto.WithPBKDF2Iterations(m.SourceIterations))
}
// Build target engine with new iterations for encryption.
if m.TargetEngine == nil && m.TargetIterations > 0 {
    m.TargetEngine, err = crypto.NewEngineWithOpts(password, nil,
        crypto.WithPBKDF2Iterations(m.TargetIterations))
}
```

**Note:** because `engine.deriveKeyWithParams` reads iterations from the
`MetaKDFParams` field on decrypt, the `SourceEngine` does **not** need to be
constructed with the source iteration count for correct decryption — it will
read it from each object's metadata (or fall back to 100k when absent). The
`SourceIterations` / `TargetIterations` fields are therefore primarily
documentation and for the edge case where the caller wants the migrator to
auto-construct engines from a password.

### 2.8 argon2id Compile-Time Gates

Two stub files using the existing `//go:build` pattern:

**`internal/crypto/kdf_argon2_default.go`** (`//go:build !fips`):

```go
//go:build !fips

package crypto

import "errors"

// ErrArgon2NotEnabled is returned when argon2id is parsed from metadata but
// the gateway was not compiled with argon2id support enabled.
// Future: replace body with real argon2id implementation.
var ErrArgon2NotEnabled = errors.New("argon2id KDF is not enabled in this build")

func deriveKeyArgon2id(_ []byte, _ KDFParams) ([]byte, error) {
    return nil, ErrArgon2NotEnabled
}
```

**`internal/crypto/kdf_argon2_fips.go`** (`//go:build fips`):

```go
//go:build fips

package crypto

func deriveKeyArgon2id(_ []byte, _ KDFParams) ([]byte, error) {
    return nil, ErrAlgorithmNotApproved
}
```

`deriveKeyWithParams` in `kdf.go` (or `engine.go`) calls `deriveKeyArgon2id` for
the `argon2id` case. This ensures both files compile cleanly in both modes, and
the future argon2id implementation only needs to replace the body of
`deriveKeyArgon2id` in the `!fips` file.

## 3. Metadata Backward Compatibility Matrix

| Object written by | `MetaKDFParams` present? | Decrypt behaviour |
|---|---|---|
| Pre-V1.0-SEC-H03 gateway (any version) | No | `ParseKDFParams("")` returns `{PBKDF2-SHA256, 100000}` (legacy fallback) |
| V1.0-SEC-H03+ gateway, `iterations=600000` | Yes: `pbkdf2-sha256:600000` | Parsed and used directly |
| V1.0-SEC-H03+ gateway, `iterations=100000` (explicit) | Yes: `pbkdf2-sha256:100000` | Parsed and used directly |
| Future argon2id gateway | Yes: `argon2id:2:19456:1` | Routed to `deriveKeyArgon2id`; fails with `ErrArgon2NotEnabled` on non-argon2id builds |

## 4. Migration Runbook (s3eg-migrate --filter=kdf)

Document to be added to `docs/MIGRATION.md` section **"Upgrading KDF parameters"**:

```
1. Verify the new gateway is running with encryption.kdf.pbkdf2.iterations: 600000.
2. Run a dry-run scan to count affected objects:
     s3eg-migrate \
       --bucket my-bucket \
       --filter kdf \
       --dry-run \
       --state-file /tmp/kdf-migration.json

3. When satisfied, run the actual migration:
     s3eg-migrate \
       --bucket my-bucket \
       --filter kdf \
       --state-file /tmp/kdf-migration.json \
       --workers 8 \
       --verify

4. Optionally rerun with --dry-run to confirm 0 ClassD objects remain.
```

Because `deriveKeyWithParams` reads iterations from metadata on decrypt, the
source engine automatically handles both 100k and 600k objects during the
migration window. No separate "source-iterations" flag is strictly required —
it is provided only for documentation clarity.

### 5.10 Harness changes (`test/harness/options.go` and `test/harness/gateway.go`)

To support the new conformance tests, two additions are needed:

#### `options.go` — new field and option

```go
type options struct {
    // ... existing fields ...
    pbkdf2Iterations int // 0 means use default (600000)
}

// WithPBKDF2Iterations sets the PBKDF2 iteration count used by the gateway.
// If not set, the gateway default (600000) is used.
func WithPBKDF2Iterations(n int) Option {
    return func(o *options) { o.pbkdf2Iterations = n }
}
```

#### `gateway.go` — pass iterations to engine and config

In `StartGateway`, after setting `cfg.Encryption.Password`:

```go
if o.pbkdf2Iterations > 0 {
    cfg.Encryption.KDF = config.KDFConfig{
        PBKDF2: config.PBKDF2Config{
            Iterations: o.pbkdf2Iterations,
        },
    }
}
```

And when constructing the `encryptionEngine`, pass the configured value:

```go
encryptionEngine, err := crypto.NewEngineWithOpts(
    []byte(o.encryptionPassword),
    compressionEngine,
    crypto.WithPBKDF2Iterations(cfg.Encryption.KDF.PBKDF2.Iterations),
)
```

This ensures the in-process gateway uses the same iteration count for both the
engine and the config, exactly as the real server does.

## 7. Open Implications / Risks

| Risk | Mitigation |
|---|---|
| Performance: 600k PBKDF2 adds ~100–300 ms CPU per encrypt/decrypt on typical hardware | Document in config; operators can tune down if latency is critical; chunked path derives key once per object, not per chunk |
| PasswordKeyManager envelope format change (4-byte prefix) | Backward compat: UnwrapKey detects old-format by length check; old envelopes (92 bytes) fall back to 100k |
| Chunked path has multiple PBKDF2-derived key points — must audit all call sites | See §2.3 — search all `deriveKey` and `pbkdf2.Key` call sites before implementation |
| KDF-params field adds one more metadata entry per object — check per-provider metadata limits | `MetadataCompactor` already handles key compaction for providers with tight limits; `MetaKDFParams` must be added to the compaction map |

### [V1.0-CRYPTO-1] argon2id KDF Support

- **Owner:** Crypto
- **Priority:** P2 (post V1.0-SEC-H03, non-FIPS deployments only)
- **Labels:** `area:crypto`, `area:security`, `enhancement`

**Summary:** Add argon2id as an optional KDF for non-FIPS deployments. The
`kdf_argon2_default.go` stub and `MetaKDFParams` format introduced by
V1.0-SEC-H03 provide the full infrastructure; this issue tracks the actual
implementation.

**Motivation:**
- argon2id is memory-hard (PHC winner 2015); OWASP #1 recommendation
- PBKDF2 is CPU-only; vulnerable to GPU/ASIC brute-force at any iteration count
- argon2id cannot be used in FIPS 140-3 builds (no validated implementation)

**Conditions for activation:**
- Operator sets `encryption.kdf.algorithm: argon2id` (new field inside `KDFConfig`)
- FIPS build (`-tags fips`) continues to return `ErrAlgorithmNotApproved`
- Migration path: `s3eg-migrate --filter=kdf` handles both PBKDF2→argon2id and argon2id→PBKDF2

**Implementation Guide:**
- [ ] Replace `ErrArgon2NotEnabled` body in `kdf_argon2_default.go` with `golang.org/x/crypto/argon2.IDKey`
- [ ] Add `Algorithm string` field to `KDFConfig` (yaml: `encryption.kdf.algorithm`); default `"pbkdf2-sha256"`
- [ ] Add `Argon2id Argon2idConfig` field to `KDFConfig` (yaml: `encryption.kdf.argon2id`) with `Time`, `Memory`, `Threads` fields
- [ ] Config validation: reject `algorithm: argon2id` in FIPS builds at startup
- [ ] Wire `cfg.Encryption.KDF.Algorithm` into engine so it selects the correct KDF on encrypt
- [ ] Add argon2id KAT tests; update `docs/MIGRATION.md` with argon2id migration section

**Definition of Done:**
- [ ] Non-FIPS: encrypt with argon2id; decrypt; plaintext matches
- [ ] FIPS: `encryption.kdf.algorithm: argon2id` rejected at startup with `ErrAlgorithmNotApproved`
- [ ] Migration tool handles PBKDF2→argon2id and argon2id→PBKDF2 re-encryption
