# Milestone v0.6 — Issue Tracker and Implementation Guide

This document tracks all issues for v0.6 with implementation guides, tests, and acceptance criteria derived from `docs/ROADMAP.md`.

## Scope and Goals
- Security/Crypto: Pluggable KeyManager, optional FIPS-compliant crypto build profile
- S3 Compatibility: Multipart copy; Object Lock/Retention passthrough guidance
- Performance: Zero-copy streaming paths where feasible; retry policy customization
- Observability: pprof endpoints gated behind admin flag; profiling recipes
- Operations & Helm/K8s: Blue/green and canary examples; values validation/schema
- Config & UX: Admin API for safe key rotation (drain + cutover)
- Testing & Quality: Performance baselines; coverage gate >=80%; mutation testing on crypto-free critical code
- Docs: Migration guide for enabling KMS mode from single-password

## Milestone Acceptance Criteria
- Coverage gate (>=80%) and mutation testing on critical crypto-free code.
- Helm values validation (schema/JSONSchema) and default overlays.

Labels: `milestone:v0.6`, `area:kms`, `area:s3`, `area:perf`, `area:observability`, `area:helm`, `area:testing`.

## [V0.6-SEC-2] FIPS-Compliant Crypto Profile (Optional) ✅ DONE
- Owner: Crypto
- Priority: P2
- Status: Completed 2026-04-17
- Plan: `docs/plans/V0.6-SEC-2-plan.md`
- ADR: `docs/adr/0005-fips-crypto-profile.md`
- Guide: `docs/FIPS.md`

### Summary
Provide optional FIPS build profile; document constraints.

### Implementation Guide
- [x] `fips` build tag added; mirrors the `hsm` / `!hsm` split.
- [x] Algorithm registry split (`algorithms_default.go` / `algorithms_fips.go`); ChaCha20-Poly1305 excluded from FIPS build.
- [x] Cipher factory split (`createaead_default.go` / `createaead_fips.go`); FIPS build returns `ErrAlgorithmNotApproved` for ChaCha20 at the API boundary.
- [x] PBKDF2 migrated unconditionally from `golang.org/x/crypto/pbkdf2` to stdlib `crypto/pbkdf2` (Go 1.24+); KAT regression test locks the output vector.
- [x] Startup `crypto.AssertFIPS()` fails closed if the runtime FIPS module is not active.
- [x] Prometheus `gateway_fips_mode` gauge and structured log banner.
- [x] `Dockerfile.fips` (Debian/distroless) with `GOFIPS140=v1.0.0` baked in.
- [x] `helm/s3-encryption-gateway/values.fips.yaml` overlay uses chart `extraEnv` to inject `GOFIPS140` / `GODEBUG`; `helm lint` passes.
- [x] `make build-fips` / `make test-fips` targets.

### Tests
- [x] `TestChaCha20Rejected` (fips tag): ChaCha20 returns `ErrAlgorithmNotApproved`.
- [x] `TestAESGCMApproved` (fips tag): AES-256-GCM round-trip succeeds.
- [x] `TestFIPSRuntimeEnabled` / `TestAssertFIPSSucceeds` (fips tag).
- [x] `TestChaCha20Registered` / `TestFIPSDisabled` / `TestAssertFIPSSucceedsInDefault` (default tag).
- [x] `TestPBKDF2KAT` + `TestPBKDF2StringPassword` (both builds).
- [x] Full crypto suite green under `-race` in both builds; full project short suite green in both builds.

### Definition of Done
- [x] Build profile documented and verifiable
- [x] `go build ./...` and `go build -tags=fips ./...` both succeed; `go vet -tags=fips` clean
- [x] `helm lint` passes with the FIPS overlay; rendered Deployment carries `GOFIPS140` env var
